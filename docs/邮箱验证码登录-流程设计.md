# 邮箱验证码登录 — 流程设计文档

## 1. 概述

新增一种登录方式：用户输入邮箱 → 接收 6 位验证码 → 输入验证码完成登录。适用于不方便输入密码的场景（如 TV、Kiosk 终端），或作为无密码登录的可选方案。

### 设计目标

- 复用现有的 EmailService（storeCode / verifyCode / sendVerificationEmail）
- 复用现有的 TokenService（generateTokens）
- 复用现有的登录后 Token 下发逻辑（setRefreshTokenCookie）
- 防止用户枚举（与之前修复的 email.js 一致）
- 添加独立的速率限制

## 3. 后端接口

### 3.1 POST /api/auth/send-login-code

发送邮箱验证码。

**请求体：**

```json
{
  "email": "user@example.com"
}
```

**处理逻辑：**

```
1. 校验 email 格式 → 失败返回 400
2. 查找用户 findByEmail(email)
3. 仅当用户存在、已启用、email_verified=true 时才实际生成并发送验证码
4. 始终返回泛化响应（防枚举）：
   → 200 { message: "如果该邮箱已注册，验证码已发送", expires_in: 600 }
5. 验证码有效期：600 秒（10 分钟），类型标记为 'login'
```

**速率限制：**

| 维度 | 限制 | 窗口 |
|------|------|------|
| IP | 5 次 | 1 小时 |
| 邮箱 | 3 次 | 24 小时 |

复用已有的 `createEmailRateLimiter()`。

### 3.2 POST /api/auth/login-with-code

使用验证码登录。

**请求体：**

```json
{
  "email": "user@example.com",
  "code": "A3F8K2"
}
```

**处理逻辑：**

```
1. 校验 email + code 必填 → 失败返回 400
2. verifyCode(email, code, 'login')
   → 失败返回 400 { error: 'invalid_code', message: '验证码错误或已过期' }
3. 查找用户 findByEmail(email)
   → 不存在返回 401（正常流程不会发生，因为验证码发送前提是用户存在）
4. 检查 user.enabled
   → 已禁用返回 403 { error: 'account_disabled' }
5. 记录登录 IP、登录位置
6. generateTokens(user) → access_token, refresh_token, id_token
7. setRefreshTokenCookie(res, refreshToken)
8. 记录活动日志 log(user.id, ACTION.LOGIN, { method: 'email_code' })
9. 返回：
   → 200 {
       access_token, token_type: 'Bearer',
       expires_in, id_token,
       user: { id, username, email, display_name, picture, role }
     }
```

**速率限制：**

| 维度 | 限制 | 窗口 |
|------|------|------|
| IP | 10 次 | 15 分钟 |

复用 `loginLimiter`（与密码登录同级别限制）。

## 5. 安全设计

### 5.1 防止用户枚举

发送验证码接口始终返回相同的成功消息，不区分邮箱是否存在、是否已验证。与 [src/routes/email.js](file:///d:/phpstudy_pro/WWW/zjsso/src/routes/email.js) 的修复策略一致。

### 5.2 验证码安全

| 项目 | 规格 |
|------|------|
| 字符集 | 6 位字母数字（0-9, A-Z），36 字符空间 |
| 有效期 | 10 分钟 |
| 存储方式 | Redis，SHA256 哈希存储 |
| 一次性 | verifyCode() 验证成功后自动删除 |
| 发送限制 | 每 IP 5 次/小时，每邮箱 3 次/24 小时 |

验证码生成复用 `emailService.generateCode()`，验证码类型为 `'login'`。

### 5.3 速率限制

| 接口 | 限制器 | 参数 |
|------|--------|------|
| send-login-code | createEmailRateLimiter() | IP: 5/h, Email: 3/天 |
| login-with-code | createRateLimiter (loginLimiter) | IP: 10/15min |

### 5.4 账号状态检查

仅当用户存在且 `enabled = true` 且 `email_verified = true` 时才发送验证码。被禁用的账号不会收到验证码通知。

## 8. 测试要点

1. **正常流程**：输入已注册邮箱 → 收到验证码 → 输入正确验证码 → 登录成功
2. **错误验证码**：输入错误验证码 → 提示"验证码错误或已过期"
3. **过期验证码**：等待 10 分钟后使用 → 提示"验证码错误或已过期"
4. **未注册邮箱**：发送时返回泛化成功消息（不泄露邮箱是否存在）
5. **未验证邮箱**：不会收到验证码（与未注册邮箱同等对待）
6. **被禁用账号**：不会收到验证码
7. **速率限制**：同一 IP 频繁发送 → 429；同一邮箱频繁发送 → 429
8. **验证码一次性**：同一验证码第二次使用 → 失败
