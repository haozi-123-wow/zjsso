# OIDC 单点登录系统 - 开发计划

## 项目概述

基于 **Node.js + MySQL 5.7 + Redis** 构建的 OIDC（OpenID Connect）单点登录系统，同时支持 **有状态（Stateful）** 和 **无状态（Stateless）** 两种会话模式，前端使用 Vue.js 开发。

---

|------|------|------|------|

| Web框架 | Express | 4.x | HTTP服务 |
| 数据库 | MySQL | 5.7+ | 持久化存储 |
| 缓存 | Redis | 6.x+ | 会话管理/速率限制 |
| Token | jsonwebtoken | 9.x | JWT签发与验证 |
| ORM | mysql2 | 3.x | MySQL驱动（直接SQL） |
| 密码加密 | bcryptjs | 2.x | 密码哈希 |
| 参数校验 | express-validator | 7.x | 请求参数校验 |

**重点:** 这是系统的核心设计——双模式会话管理。

#### 有状态模式流程
```
用户登录 → 生成session_id → 存Redis(含用户信息) →
返回session_id作为token → 验证时查Redis → TTL自动过期
```

#### 无状态模式流程
```
用户登录 → 生成JWT(含用户claims和jti) →
返回JWT作为token → 验证时仅验签(+查黑名单) →
登出时将jti加入Redis黑名单
```

**详细说明:**

#### 4.1 授权服务 (Authorization)
- 处理 `/oauth/authorize` 逻辑
- 验证client_id、redirect_uri、scope
- PKCE code_challenge验证
- 生成授权码（支持state和nonce）
- 管理OIDC会话状态流转

#### 4.2 Token服务 (Token)
- 处理 `/oauth/token` 逻辑
- 授权码换token（验证code、PKCE code_verifier）
- refresh_token换新token（轮换机制）
- client_credentials授权
- 根据SESSION_MODE选择签发JWT或随机字符串
- 生成id_token（含sub/iss/aud/exp/nonce等标准claims）

#### 4.3 UserInfo服务
- 处理 `/userinfo` 请求
- 验证access_token
- 根据scope返回用户claims
- 支持claims端点

#### 4.4 Provider主入口
- OpenID Configuration发现文档
- JWKS端点路由
- 统一错误处理
- 日志记录

## 核心算法细节

### 1. 授权码生成

```
code = base64url( random_bytes(32) )
```

### 2. PKCE Code Challenge 生成

```
code_verifier = base64url( random_bytes(32) )
code_challenge = base64url( SHA256(code_verifier) )
```

### 3. JWT ID Token 结构（无状态模式）

```json
{
  "iss": "http://localhost:3000",
  "sub": "user-uuid",
  "aud": "client-id",
  "exp": 1735689600,
  "iat": 1735686000,
  "auth_time": 1735686000,
  "nonce": "random-nonce-value",
  "acr": "urn:mace:incommon:iap:silver",
  "amr": ["pwd"],
  "at_hash": "base64url(half(SHA256(access_token)))"
}
```

### 4. Token轮换机制

```
旧refresh_token
  → 验证 hash(token) 在数据库中存在且未使用、未撤销
  → 标记旧refresh_token为 used=true
  → 生成新 refresh_token
  → 新 refresh_token 存入数据库
  → 返回新 refresh_token
```

## Redis Key 命名规范

| Key模式 | 用途 | TTL |
|---------|------|-----|
| `session:{session_id}` | 有状态会话 | SESSION_EXPIRES_IN |
| `token:{token_hash}` | 有状态access_token | JWT_EXPIRES_IN |
| `blacklist:jti:{jti}` | JWT黑名单 | 剩余有效期 |
| `blacklist:refresh:{hash}` | refresh_token黑名单 | 剩余有效期 |
| `ratelimit:{path}:{ip}:{window}` | 速率限制 | 窗口时间 |
| `auth_code:{code}` | 授权码缓存 | 300秒 |
| `oauth:{provider}:{state}` | 第三方OAuth临时状态 | 600秒 |
| `webauthn:challenge:{uid}:{sid}` | WebAuthn挑战码 | 300秒 |
| `email:code:{email}` | 邮件验证码 | 3600秒 |
| `email:attempt:{email}` | 邮件发送次数限制 | 86400秒 |
| `geetest:challenge:{challenge}` | 极验挑战码 | 300秒 |
| `geetest:validate:{seccode}` | 极验二次校验 | 86400秒 |

## 部署指南

### 开发环境启动

```bash
# 1. 确保MySQL和Redis运行中

# 2. 安装依赖
npm install

# 3. 配置.env文件
cp .env.example .env  # 修改数据库连接信息

# 4. 运行数据库迁移
npm run migrate

# 5. 填充测试数据
npm run seed

# 6. 启动服务
npm run dev
```

### 生产环境注意事项

1. **JWT_SECRET** 必须使用强随机密钥（建议RSA非对称密钥）
2. **SESSION_SECRET** 必须修改
3. 启用Helmet安全中间件
4. 配置CORS白名单
5. 数据库定期备份
6. Redis配置密码和持久化
