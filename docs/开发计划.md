# OIDC 单点登录系统 - 开发计划

## 项目概述

基于 **Node.js + MySQL 5.7 + Redis** 构建的 OIDC（OpenID Connect）单点登录系统，同时支持 **有状态（Stateful）** 和 **无状态（Stateless）** 两种会话模式，前端使用 Vue.js 开发。

---

## 技术栈

| 层级 | 技术 | 版本 | 用途 |
|------|------|------|------|
| 后端运行时 | Node.js | 18+ | 服务端运行环境 |
| Web框架 | Express | 4.x | HTTP服务 |
| 数据库 | MySQL | 5.7+ | 持久化存储 |
| 缓存 | Redis | 6.x+ | 会话管理/速率限制 |
| Token | jsonwebtoken | 9.x | JWT签发与验证 |
| ORM | mysql2 | 3.x | MySQL驱动（直接SQL） |
| 密码加密 | bcryptjs | 2.x | 密码哈希 |
| 参数校验 | express-validator | 7.x | 请求参数校验 |

#### 有状态模式流程
```
用户登录 → 生成JWT + 存入Redis(token:accessToken) →
验证时查Redis → TTL自动过期 → 登出时删除Redis key
```

#### 无状态模式流程
```
用户登录 → 生成JWT(含user claims和jti) →
返回JWT作为token → 验证时仅验签(+查Redis黑名单) →
登出时将jti加入Redis黑名单
```

## 核心算法细节

### 1. 授权码生成

```
code = base64url( random_bytes(32) )
```

### 2. PKCE Code Challenge 生成

```
code_verifier = base64url( random_bytes(32) )
code_challenge = base64url( SHA256(code_verifier) )
```

### 3. JWT ID Token 结构（已实现）

```json
{
  "iss": "http://localhost:3000",
  "sub": "user-uuid",
  "aud": "zjsso",
  "exp": 1735689600,
  "iat": 1735686000,
  "name": "display_name",
  "preferred_username": "username",
  "email": "user@example.com",
  "email_verified": true,
  "picture": "avatar-url",
  "locale": "zh-CN",
  "qq": "qq-number"
}
```

### 4. Token轮换机制（已实现）

```
旧refresh_token
  → SHA256哈希 → 查refresh_tokens表(未使用、未撤销、未过期)
  → 标记旧refresh_token为 used=true
  → 生成新refresh_token(uuid v4)
  → 存入refresh_tokens表
  → 返回新refresh_token
```
